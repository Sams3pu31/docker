1. Что такое Kubernetes и для чего его используют?

Kubernetes — это система оркестрации контейнеров с открытым исходным кодом, созданная Google и
переданная Cloud Native Computing Foundation (CNCF).
Kubernetes используется для автоматизации развертывания, масштабирования и управления контейнеризированными приложениями.
Он позволяет управлять контейнерными кластерами в различных средах, включая локальные, облачные и гибридные инфраструктуры.

2. Как связаны Kubernetes и контейнеры?

Kubernetes работает с контейнерами, такими как Docker, для развертывания, управления и масштабирования приложений.
Контейнеры изолируют приложения и их зависимости, обеспечивая согласованную среду выполнения.
Kubernetes управляет жизненным циклом этих контейнеров, распределяя их по узлам кластера, обеспечивая отказоустойчивость,
автоматическое восстановление, балансировку нагрузки и масштабирование.

3. Какие преимущества дает использование Kubernetes для управления приложениями?

Использование Kubernetes для управления приложениями дает несколько преимуществ:

   - Автоматизация развертывания и управления: Kubernetes автоматизирует развертывание и управление контейнеризированными приложениями.
   - Масштабируемость:Kubernetes автоматически масштабирует приложения в зависимости от нагрузки.
   - Высокая доступность: Kubernetes обеспечивает отказоустойчивость и восстановление приложений в случае сбоя.
   - Управление конфигурациями: Kubernetes позволяет легко управлять конфигурациями приложений через ConfigMaps и Secrets.
   - Портативность:Kubernetes обеспечивает согласованную среду выполнения на различных инфраструктурах.
   - Балансировка нагрузки и сервис-дискавери:Kubernetes автоматически распределяет трафик между контейнерами и
   обеспечивает их обнаружение.

4. **Как Kubernetes масштабирует приложения?**

Kubernetes масштабирует приложения с помощью механизма реплик. Реплика управляет количеством экземпляров (реплик)
подов, которые должны работать в любой момент времени.

6. **Как сконфигурировать деплой в Kubernetes?**

Для конфигурирования деплоя в Kubernetes создается манифест YAML файл, который описывает желаемое состояние деплоймента. Пример манифеста деплоймента:

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-container
        image: my-image:latest
        ports:
        - containerPort: 8080

Примените деплоймент с помощью команды:

kubectl apply -f deployment.yaml


Этот манифест создает деплоймент с тремя репликами подов,
содержащих контейнеры с указанным образом.
Kubernetes будет поддерживать заданное количество реплик и автоматически управлять их состоянием.
